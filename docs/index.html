Feature set

Built like a real research workflow, not a single prompt wrapper.

01

Multi-agent pipeline

Planner, search, reader, synthesis, and citation auditor stages create a traceable research process.

02

Live search providers

Adapters support Tavily, Brave Search, Bing Web Search, SerpAPI, and a no-key Wikipedia fallback.

03

Cited synthesis

Answers are rendered with numbered source citations and a source panel for quick evidence review.

04

Streaming UX

The UI receives newline-delimited JSON events so users can see progress while agents work.

05

Research history

Completed runs are stored locally and can be reopened from the sidebar.

06

Markdown export

Finished briefs can be exported for notes, sharing, or portfolio demos.

Architecture

Simple Node backend, static frontend, provider-based search layer.

The project avoids framework overhead while still demonstrating production-shaped concerns: provider abstraction, streaming responses, history persistence, source scoring, environment-based configuration, and a clean browser UI.

  1. Planner AgentTurns a user question into targeted search tasks.
  2. Search AgentQueries the configured real-time web search provider.
  3. Reader AgentFetches readable page text and enriches source metadata.
  4. Synthesis AgentWrites a concise answer from the gathered sources.
  5. Citation AuditorVerifies cited source IDs before rendering the result.

Run locally

Clone, configure, and start the app.

git clone https://github.com/risingsummit/multi-agent-ai-web-researcher.git
cd multi-agent-ai-web-researcher
copy .env.example .env
npm start

Add a search API key for live web providers, or use the built-in Wikipedia fallback for a no-key demo.
